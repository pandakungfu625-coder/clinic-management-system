What is this report?

This view is not stored in the database. It is generated by combining data from multiple tables.

  • Students — who the learner is
  • Courses — what is being taught
  • Enrollments — how students and courses are linked

The system dynamically joins these tables using SQL JOIN operations to produce a meaningful report.

Data flow: students → enrollments → courses

SQL used to generate this report (JOIN)

Read-only 
SELECT
    e.id AS enrollment_id,
    s.name AS student_name,
    c.title AS course_title,
    e.enrolled_on
FROM enrollments e
JOIN students s ON e.student_id = s.id
JOIN courses c ON e.course_id = c.id;
Note: Course information is not duplicated in the student table. This follows database normalization principles.
